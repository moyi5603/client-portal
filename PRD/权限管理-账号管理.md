# 账号管理 - 账号管理

## 1. 概述

### 1.1 功能定位
账号管理模块是Client Portal权限管理系统的核心功能之一，用于管理系统中的用户账号。主账号可以通过此模块创建、编辑、删除客户子账号和Partner账号，并为账号分配Customer访问权限和角色。

### 1.2 业务价值
- **统一管理**：集中管理租户下的所有账号，提高管理效率
- **权限控制**：通过分配Customer和角色，实现细粒度的数据访问控制
- **灵活配置**：支持多种账号类型和状态，满足不同业务场景需求
- **数据隔离**：通过Customer分配实现多租户数据隔离

### 1.3 目标用户
- **主账号（MAIN）**：拥有账号管理的完整权限，可以创建、编辑、删除所有类型的账号
- **拥有账号管理权限的账号**：通过角色分配获得账号管理权限的账号，可以创建、编辑、删除账号（但不能管理自己）

## 2. 功能需求

### 2.1 账号列表展示

#### 2.1.1 列表字段
- **用户名**：账号的唯一标识（必填，唯一）
- **邮箱**：账号邮箱地址（必填，唯一，需符合邮箱格式）
- **手机号**：账号手机号码（可选）
- **账号类型**：显示账号类型
  - 主账号（MAIN）
  - 客户子账号（CUSTOMER）
  - Partner账号（PARTNER）
- **可访问Customer**：显示账号可访问的Customer列表
  - 主账号：显示【全部customer】（注：此处的"全部customer"是指Central系统中分配给该主账号的所有Customer，对该主账号而言即为"全部"）
  - 各子账号：显示可访问Customer（accessibleCustomerIds）
  - 显示格式：`Customer名称(Customer代码)`
  - 无Customer时显示"无"
- **角色信息**：显示账号分配的角色列表
  - 主账号：显示【全部权限】（注：Central系统分配给该主账号的权限即为该主账号的"全部权限"）
  - 子账号：显示已分配的角色名称，多个角色自动换行显示
  - 无角色时显示"无"
- **状态**：账号当前状态
  - 启用（ACTIVE）
  - 禁用（INACTIVE）
  - 暂停（SUSPENDED）
- **操作**：编辑、删除按钮
  - 主账号：不支持任何操作，显示“-”
  - 子账号：支持编辑、删除操作

#### 2.1.2 列表功能
- **筛选功能**：支持按以下条件进行筛选
  - **用户名**：支持模糊搜索（不区分大小写）
  - **邮箱**：支持模糊搜索（不区分大小写）
  - **手机号**：支持模糊搜索
  - **可访问Customer**：支持多选Customer进行筛选
    - 使用AND逻辑：账号必须包含所有选中的Customer才会显示
    - 主账号：不受Customer筛选影响（始终显示）
  - **账号类型**：支持选择主账号、客户子账号、Partner账号
  - **状态**：支持选择启用、禁用、暂停
  - **组合筛选**：多个筛选条件可以同时使用，筛选结果为所有条件的交集
- **分页显示**：每页10条记录，支持翻页

### 2.2 创建账号

#### 2.2.1 必填字段
- **账号类型**：CUSTOMER 或 PARTNER
- **用户名**：在同一租户内唯一
- **邮箱**：有效的邮箱地址，在同一租户内唯一
- **密码**：创建账号时的初始密码

#### 2.2.2 可选字段
- **手机号**：手机号码
- **状态**：默认"启用"，可选择"禁用"或"暂停"
- **可访问Customer**：可多选，为空则无Customer访问权限，只能在Central系统分配给主账号的Customer范围内选择
- **角色**：可多选，为空则无角色权限，只能选择【启用】状态的角色

#### 2.2.3 验证规则
- 用户名、邮箱在同一租户内必须唯一
- 邮箱格式验证
- 必填项信息完整

### 2.3 编辑账号

#### 2.3.1 可编辑字段
- **用户名**：需验证唯一性（排除当前账号）
- **邮箱**：需验证唯一性和格式（排除当前账号）
- **手机号**：手机号码
- **状态**：启用/禁用/暂停
- **可访问Customer**：可多选，只能在Central系统分配给主账号的Customer范围内选择
- **角色**：可多选，只能选择【启用】状态的角色
- **账号类型**：创建后不可修改
- **密码**：编辑时修改视为重置密码

### 2.4 删除账号

#### 2.4.1 删除确认
- 删除前显示确认对话框："确定要删除这个账号吗？"
- 确认后执行删除操作

#### 2.4.2 删除限制
- 只能删除当前主账号下的账号
- 删除后账号数据永久删除（需谨慎操作）

## 3. 交互流程

### 3.1 创建账号流程

```
1. 用户点击"创建账号"按钮
   ↓
2. 弹出创建账号对话框
   ↓
3. 用户完善账号信息
   ↓
4. 用户点击"确定"按钮
   ↓
5. 验证信息（邮箱格式、必填项、唯一性）
   ↓
6. 创建成功，关闭对话框，刷新列表
```

### 3.2 编辑账号流程

```
1. 用户点击账号行的"编辑"按钮
   ↓
2. 弹出编辑账号对话框，预填充账号信息
   ↓
3. 用户修改需要更新的字段
   ↓
4. 用户点击"确定"按钮
   ↓
5. 验证信息（邮箱格式、必填项、唯一性）
   ↓
6. 更新成功，关闭对话框，刷新列表
```

### 3.3 删除账号流程

```
1. 用户点击账号行的"删除"按钮
   ↓
2. 弹出确认对话框："确定要删除这个账号吗？"
   ↓
3. 用户点击"确定"
   ↓
4. 删除成功，刷新列表
```


## 4. 验收标准

### 4.1 功能验收

#### 4.1.1 列表展示
- [ ] **列表展示**：能够正确显示账号列表，包含所有必需字段（用户名、邮箱、手机号、账号类型、可访问Customer、角色信息、状态、操作）
- [ ] **主账号显示**：主账号的"可访问Customer"显示【全部customer】，"角色信息"显示【全部权限】
- [ ] **子账号显示**：子账号正确显示可访问Customer列表和角色名称，多个角色自动换行显示
- [ ] **空值显示**：无Customer或角色时显示"无"（灰色文字）
- [ ] **主账号操作限制**：主账号不支持任何操作，操作列显示"-"
- [ ] **子账号操作**：子账号支持编辑、删除操作

#### 4.1.2 创建账号
- [ ] **必填字段验证**：账号类型、用户名、邮箱、密码为必填项
- [ ] **可选字段**：手机号为可选字段，不填写不影响创建
- [ ] **唯一性验证**：用户名、邮箱在同一租户内唯一
- [ ] **格式验证**：邮箱格式验证正确
- [ ] **Customer验证**：只能在Central系统给主账号分配的Customer范围内进行选择
- [ ] **角色验证**：只能分配存在的角色，且只能选择【启用】状态的角色
- [ ] **状态默认值**：创建账号时状态默认为"启用"

#### 4.1.3 编辑账号
- [ ] **编辑功能**：能够成功编辑账号信息
- [ ] **可编辑字段**：用户名、邮箱、手机号、状态、可访问Customer、角色可以编辑
- [ ] **唯一性验证**：编辑时验证用户名、邮箱、手机号的唯一性（排除当前账号）
- [ ] **格式验证**：编辑时邮箱格式验证正确

#### 4.1.4 删除账号
- [ ] **删除确认**：删除前显示确认对话框"确定要删除这个账号吗？"
- [ ] **删除功能**：能够成功删除账号
- [ ] **删除后刷新**：删除成功后自动刷新列表

#### 4.1.5 筛选功能
- [ ] **模糊搜索**：能够按用户名、邮箱、手机号进行模糊搜索（不区分大小写）
- [ ] **Customer筛选**：能够按可访问Customer进行多选筛选
- [ ] **Customer筛选逻辑**：可访问Customer筛选使用AND逻辑（账号必须包含所有选中的Customer）
- [ ] **Customer筛选主账号**：主账号不受Customer筛选影响，始终显示
- [ ] **账号类型筛选**：能够按账号类型筛选（主账号、客户子账号、Partner账号）
- [ ] **状态筛选**：能够按状态筛选（启用、禁用、暂停）
- [ ] **组合筛选**：多个筛选条件可以组合使用（多个条件同时生效，结果为交集）
- [ ] **重置筛选**：支持重置筛选条件，一键清除所有筛选并恢复显示所有账号

#### 4.1.6 权限控制
- [ ] **访问权限**：只有主账号可以访问账号管理功能
- [ ] **租户隔离**：只能管理当前租户下的账号


### 4.2 安全验收

- [ ] **权限验证**：非主账号无法访问账号管理功能
- [ ] **租户隔离**：只能管理当前租户下的账号
- [ ] **数据验证**：所有输入数据都经过验证（前端和后端）
- [ ] **密码安全**：密码不在前端显示，后端加密存储
- [ ] **唯一性验证**：用户名、邮箱、手机号在同一租户内唯一性验证正确

## 5. 注意事项

### 5.1 业务规则

1. **账号类型不可修改**：账号创建后，账号类型不可修改
2. **主账号管理**：
   - 主账号由Central系统管理，不在本模块中创建
   - 主账号不支持任何操作（编辑、删除），操作列显示"-"
   - 主账号的"可访问Customer"固定显示【全部customer】
   - 主账号的"角色信息"固定显示【全部权限】
3. **Customer分配**：
   - 只能在Central系统分配给主账号的Customer范围内进行选择
4. **角色分配**：
   - 只能选择【启用】状态的角色
   - 已废弃（DEPRECATED）状态的角色不可选择
5. **手机号可选**：手机号为可选字段，不填写不影响账号创建和编辑
6. **唯一性验证**：用户名、邮箱在同一租户内必须唯一
