# 权限管理 - 权限查看

## 1. 概述

### 1.1 功能定位
权限查看模块是Client Portal权限管理系统的核心功能之一，用于查看和分析系统中的权限分配情况。主账号可以通过此模块以不同的视图方式查看角色权限矩阵，了解各角色在不同模块下的权限配置，以及使用各角色的用户分布情况，为权限管理决策提供数据支持。

### 1.2 业务价值
- **权限透明化**：清晰展示所有角色的权限配置，便于权限审计和管理
- **多维度分析**：支持按角色和按页面两种视图，满足不同的分析需求
- **用户分布可视化**：直观显示各角色的用户分布情况，便于了解权限使用情况
- **权限决策支持**：为权限调整和优化提供数据依据
- **合规审计**：支持权限合规检查和审计要求
- **权限优化**：通过权限使用情况分析，识别冗余或缺失的权限配置

### 1.3 目标用户
- **主账号（MAIN）**：拥有权限查看的完整权限，可以查看所有角色的权限配置
- **拥有权限查看权限的账号**：通过角色分配获得权限查看权限的账号，可以查看权限矩阵

## 2. 功能需求

### 2.1 权限矩阵展示

#### 2.1.1 视图模式
- **按所有角色（矩阵视图）**：以角色为行，模块为列的矩阵形式展示权限
  - 行：角色名称（只显示状态为ACTIVE且有权限配置的角色）
  - 列：系统模块-一级菜单（Dashboards、Purchase Management、Sales Order等13个模块）
  - 交叉点：显示该角色在该模块下拥有的操作权限图标
  - 最后一列：显示使用该角色的用户信息（头像、计数、用户名列表）
  - 固定列：角色名称列固定在左侧
  - 水平滚动：支持水平滚动查看更多模块列

- **按所有页面（列表视图）**：以权限条目为行的列表形式展示权限
  - 角色名称：权限所属的角色
  - 模块：权限所属的模块（显示模块颜色标识和名称）
  - 功能：权限所属的具体功能页面
  - 权限：该权限条目包含的操作列表（以标签形式显示）
  - 用户：使用该角色的用户信息（头像、计数、用户名列表）

#### 2.1.2 权限图例
系统提供权限图例说明，帮助用户理解权限图标的含义：
- **查看**：绿色眼睛图标，包含VIEW、INVOICE_DETAIL等操作
- **创建**：绿色加号图标，包含CREATE操作
- **编辑**：紫色编辑图标，包含EDIT、HOLD_INVENTORY、RELEASE_INVENTORY、ADD_ATTACHMENT、SET_ALERT、SET_DEFAULT、RELOAD等操作
- **删除**：红色删除图标，包含DELETE、CANCEL等操作
- **导出**：蓝色下载图标，包含EXPORT、PRINT_PACKING_SLIP、DOWNLOAD_PDF、DOWNLOAD_TEMPLATE、DOWNLOAD等操作
- **导入**：蓝色刷新图标，包含IMPORT、IMPORT_RMA、BATCH_IMPORT等操作
- **其他操作**：如PAY等特殊操作，使用绿色勾选图标

#### 2.1.3 模块配置
系统支持13个核心模块，每个模块配置不同的标识颜色：
- Dashboards（紫色）
- Purchase Management（橙色）
- Sales Order（绿色）
- Work Order（粉色）
- Inbound（红色）
- Inventory（蓝色）
- Outbound（紫色）
- Returns（橙色）
- Yard Management（绿色）
- Supply Chain Mgmt（粉色）
- Finance（红色）
- System Management（蓝色）
- Permission Management（紫色）

### 2.2 筛选和搜索功能

#### 2.2.1 筛选条件
- **模块筛选**：支持选择特定模块或"所有模块"
  - 下拉选择框，包含所有系统模块
  - 默认选择"所有模块"
  - 筛选后只显示包含该模块权限的角色或权限条目
- **角色名称筛选**：支持按角色名称进行模糊搜索
  - 文本输入框，支持实时搜索
  - 不区分大小写的模糊匹配
  - 支持清空功能

#### 2.2.2 筛选逻辑
- **组合筛选**：模块筛选和角色名称筛选可以同时使用
- **筛选结果**：筛选条件为"AND"关系（同时满足）
- **实时更新**：筛选条件变更时，表格数据实时更新
- **重置功能**：支持一键重置所有筛选条件

#### 2.2.3 筛选行为差异
- **矩阵视图**：在角色级别进行筛选，如果角色包含目标模块的权限，则显示该角色的所有权限
- **列表视图**：在权限条目级别进行筛选，只显示匹配模块的具体权限条目

### 2.3 用户信息展示

#### 2.3.1 用户列显示
- **用户头像**：显示用户头像（最多显示3个）
- **用户计数**：显示使用该角色的用户总数
- **用户名称**：显示用户名称列表（悬停提示）
- **超量显示**：超过3个用户时显示"+N"标签，悬停显示完整用户列表
- **空状态显示**：没有用户使用该角色时显示"-"

#### 2.3.2 用户信息来源
- 基于角色分配关系，动态计算每个角色的用户列表
- 实时更新用户计数和用户信息
- 只显示当前租户下的用户信息
- 通过账号的roles字段关联角色和用户

### 2.4 数据展示和分页

#### 2.4.1 表格功能
- **固定列**：角色名称列固定在左侧，用户列固定在右侧（矩阵视图）
- **水平滚动**：支持水平滚动查看更多模块列
- **排序功能**：角色名称列不支持排序（已移除）
- **响应式设计**：适配不同屏幕尺寸
- **加载状态**：数据加载时显示加载动画
- **空状态处理**：没有权限的单元格显示"-"

#### 2.4.2 分页设置
- **矩阵视图**：每页10条记录，支持调整每页显示数量
- **列表视图**：每页20条记录，支持调整每页显示数量
- **分页信息**：显示当前页范围和总记录数
- **页面跳转**：支持页面跳转和页面大小调整

### 2.5 数据导出功能

#### 2.5.1 导出功能
- **CSV导出**：支持将当前视图的权限数据导出为CSV文件
- **导出范围**：导出当前筛选条件下的所有数据
- **导出格式**：包含角色名称、模块、功能、权限、用户信息等完整数据
- **文件命名**：按照"权限矩阵_日期时间.csv"格式命名

#### 2.5.2 刷新功能
- **手动刷新**：提供刷新按钮，支持手动重新加载数据
- **自动更新**：当权限配置发生变化时，页面数据自动更新
- **加载提示**：刷新过程中显示加载状态

## 3. 交互流程

### 3.1 权限查看流程

```
1. 用户进入权限查看页面
   ↓
2. 系统加载角色和账号数据
   ↓
3. 默认显示矩阵视图，展示所有活跃角色的权限
   ↓
4. 用户可以切换视图模式（按所有角色/按所有页面）
   ↓
5. 用户可以应用筛选条件（模块、角色名称）
   ↓
6. 系统根据筛选条件更新显示结果
```

### 3.2 筛选操作流程

```
1. 用户选择筛选条件
   ├─ 选择模块筛选
   └─ 输入角色名称筛选
   ↓
2. 系统实时更新筛选结果
   ↓
3. 用户可以点击"Search"按钮应用筛选
   ↓
4. 用户可以点击"重置"按钮清空所有筛选条件
```

### 3.3 视图切换流程

```
1. 用户点击视图切换按钮
   ├─ "按所有角色"：切换到矩阵视图
   └─ "按所有页面"：切换到列表视图
   ↓
2. 系统重新组织数据并更新显示
   ↓
3. 保持当前的筛选条件
   ↓
4. 更新分页设置（矩阵视图10条/页，列表视图20条/页）
```

### 3.4 数据导出流程

```
1. 用户点击"Export CSV"按钮
   ↓
2. 系统收集当前筛选条件下的所有数据
   ↓
3. 生成CSV文件（包含完整的权限信息）
   ↓
4. 浏览器下载文件（文件名：权限矩阵_日期时间.csv）
```

### 3.5 用户信息查看流程

```
1. 用户悬停在用户头像或用户计数上
   ↓
2. 显示完整的用户名称列表
   ↓
3. 如果用户数量超过3个，显示"+N"标签
   ↓
4. 悬停在"+N"标签上显示剩余用户列表
```

## 4. 技术规格

### 4.1 数据来源

#### 4.1.1 角色数据
- **数据源**：`/api/roles` 接口
- **筛选条件**：只显示状态为ACTIVE的角色
- **权限过滤**：只显示有权限配置的角色（permissions.length > 0）
- **实时计算**：动态计算角色的用户使用计数
- **数据结构**：
  ```typescript
  interface Role {
    id: string;
    name: string;
    description?: string;
    status: 'ACTIVE' | 'DEPRECATED';
    permissions: Permission[];
    usageCount: number;
  }
  ```

#### 4.1.2 账号数据
- **数据源**：`/api/accounts` 接口
- **关联逻辑**：通过角色ID关联角色和账号
- **用户筛选**：基于账号的roles字段筛选使用特定角色的用户
- **数据结构**：
  ```typescript
  interface Account {
    id: string;
    username: string;
    roles: string[];
  }
  ```

#### 4.1.3 权限数据
- **数据结构**：
  ```typescript
  interface Permission {
    module: string;
    page: string;
    pageCode: string;
    operations: string[];
  }
  ```

#### 4.1.4 模块配置
- **模块列表**：13个系统模块（Dashboards、Purchase Management等）
- **模块颜色**：每个模块配置不同的标识颜色
- **动态加载**：模块列表基于实际权限数据动态生成
- **配置结构**：
  ```typescript
  const MODULE_CONFIG: Record<string, { name: string; color: string }> = {
    'DASHBOARDS': { name: 'Dashboards', color: '#722ed1' },
    'PURCHASE_MANAGEMENT': { name: 'Purchase Management', color: '#fa8c16' },
    // ... 其他模块配置
  };
  ```

### 4.2 权限图标映射

#### 4.2.1 操作分类
```typescript
// 查看类操作
VIEW, INVOICE_DETAIL → 绿色眼睛图标 (#52c41a)

// 创建类操作  
CREATE → 绿色加号图标 (#52c41a)

// 编辑类操作
EDIT, HOLD_INVENTORY, RELEASE_INVENTORY, ADD_ATTACHMENT, 
SET_ALERT, SET_DEFAULT, RELOAD → 紫色编辑图标 (#722ed1)

// 删除类操作
DELETE, CANCEL → 红色删除图标 (#f5222d)

// 导出类操作
EXPORT, PRINT_PACKING_SLIP, DOWNLOAD_PDF, 
DOWNLOAD_TEMPLATE, DOWNLOAD → 蓝色下载图标 (#1890ff)

// 导入类操作
IMPORT, IMPORT_RMA, BATCH_IMPORT → 蓝色刷新图标 (#1890ff)

// 特殊操作
PAY → 绿色勾选图标 (#52c41a)
```

#### 4.2.2 图标合并逻辑
- 相同类别的操作合并显示为一个图标
- 避免重复显示相同类型的权限图标
- 提供悬停提示显示具体的操作类型
- 图标配置结构：
  ```typescript
  const OPERATION_ICONS: Record<string, { icon: React.ReactNode; color: string }> = {
    'VIEW': { icon: <EyeOutlined />, color: '#52c41a' },
    'CREATE': { icon: <PlusOutlined />, color: '#52c41a' },
    // ... 其他操作配置
  };
  ```

### 4.3 数据处理逻辑

#### 4.3.1 矩阵视图数据处理
- **行数据**：以角色为单位组织数据
- **列数据**：以模块为单位组织权限
- **交叉数据**：计算角色在特定模块下的所有操作权限
- **用户关联**：动态计算每个角色的用户列表

#### 4.3.2 列表视图数据处理
- **行数据**：以权限条目为单位组织数据
- **筛选逻辑**：在权限条目级别进行模块筛选
- **数据展开**：将角色权限展开为多个权限条目

#### 4.3.3 筛选处理逻辑
- **角色筛选**：基于角色名称的模糊匹配
- **模块筛选**：基于权限模块的精确匹配
- **组合筛选**：多个筛选条件的AND逻辑
- **实时筛选**：输入变化时立即更新结果

### 4.4 性能优化

#### 4.4.1 数据加载优化
- **批量加载**：同时加载角色和账号数据
- **缓存机制**：缓存已加载的数据，避免重复请求
- **分页加载**：大数据量时采用分页加载策略

#### 4.4.2 渲染优化
- **虚拟滚动**：大数据量时使用虚拟滚动提升性能
- **懒加载**：按需加载用户头像和详细信息
- **防抖处理**：搜索输入使用防抖机制减少请求频率

## 6. 注意事项

### 6.1 业务规则

1. **数据范围**：
   - 只显示状态为ACTIVE的角色
   - 只显示有权限配置的角色（permissions.length > 0）
   - 只显示当前租户下的数据
   - 用户信息基于角色分配关系动态计算

2. **权限显示**：
   - 权限图标按照操作类型合并显示
   - 相同类型的操作只显示一个图标
   - 提供权限图例帮助用户理解
   - 空权限单元格显示"-"

3. **筛选逻辑**：

   - 矩阵视图适合整体权限概览
   - 列表视图适合详细权限分析
   - 两种视图数据来源一致但展示方式不同
