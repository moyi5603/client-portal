# 权限管理 - 权限查看

## 1. 概述

### 1.1 功能定位
权限查看模块是Client Portal权限管理系统的核心功能之一，用于查看和分析系统中的权限分配情况。主账号可以通过此模块以不同的视图方式查看角色权限列表，了解各角色在不同模块下的权限配置，以及使用各角色的用户分布情况，为权限管理决策提供数据支持。

### 1.2 业务价值
- **权限透明化**：清晰展示所有角色的权限配置，便于权限审计和管理
- **多维度分析**：支持按角色和按页面两种视图，满足不同的分析需求
- **用户分布可视化**：直观显示各角色的用户分布情况，便于了解权限使用情况
- **权限决策支持**：为权限调整和优化提供数据依据
- **合规审计**：支持权限合规检查和审计要求
- **权限优化**：通过权限使用情况分析，识别冗余或缺失的权限配置

### 1.3 目标用户
- **主账号（MAIN）**：拥有权限查看的完整权限，可以查看所有角色的权限配置
- **拥有权限查看权限的账号**：通过角色分配获得权限查看权限的账号，可以查看权限列表

## 2. 功能需求

### 2.1 权限列表展示

#### 2.1.1 视图模式
- **按所有角色（矩阵视图）**：以角色为行，模块为列的矩阵形式展示权限
  - 行：角色名称（只显示状态为ACTIVE且有权限配置的角色）
  - 列：系统模块-一级菜单（Dashboards、Purchase Management、Sales Order等13个模块）
  - 交叉点：显示该角色在该模块下拥有的操作权限图标
  - 最后一列：显示使用该角色的用户信息（头像、计数、用户名列表）
  - 固定列：角色名称列固定在左侧
  - 水平滚动：支持水平滚动查看更多模块列

- **按所有页面（列表视图）**：以权限条目为行的列表形式展示权限
  - 角色名称：权限所属的角色
  - 模块：权限所属的模块（显示模块名称，不显示颜色标识）
  - 功能：权限所属的具体功能页面
  - 权限：该权限条目包含的操作列表（以标签形式显示）
  - 用户：使用该角色的用户信息（头像、计数、用户名列表）

#### 2.1.2 权限图例
系统提供权限图例说明，帮助用户理解权限图标的含义：
- **查看**：绿色眼睛图标，包含VIEW、INVOICE_DETAIL等操作
- **创建**：绿色加号图标，包含CREATE操作
- **编辑**：紫色编辑图标，包含EDIT、HOLD_INVENTORY、RELEASE_INVENTORY、ADD_ATTACHMENT、SET_ALERT、SET_DEFAULT、RELOAD等操作
- **删除**：红色删除图标，包含DELETE、CANCEL等操作
- **导出**：蓝色下载图标，包含EXPORT、PRINT_PACKING_SLIP、DOWNLOAD_PDF、DOWNLOAD_TEMPLATE、DOWNLOAD等操作
- **导入**：蓝色刷新图标，包含IMPORT、IMPORT_RMA、BATCH_IMPORT等操作
- **其他操作**：如PAY等特殊操作，使用绿色勾选图标

#### 2.1.3 模块配置
系统支持13个核心模块：
- Dashboards
- Purchase Management
- Sales Order
- Work Order
- Inbound
- Inventory
- Outbound
- Returns
- Yard Management
- Supply Chain Mgmt
- Finance
- System Management
- Permission Management


### 2.2 筛选和搜索功能

#### 2.2.1 筛选条件
- **模块筛选**：支持选择特定模块或"所有模块"
  - 下拉选择框，包含所有系统模块（一级菜单）
  - 默认选择"所有模块"
  - 筛选后只显示包含该模块权限的角色或权限条目
- **角色名称筛选**：支持按角色名称进行模糊搜索
  - 文本输入框，支持实时搜索
  - 不区分大小写的模糊匹配
  - 支持重置功能

#### 2.2.2 筛选逻辑
- **组合筛选**：模块筛选和角色名称筛选可以同时使用
- **筛选结果**：筛选条件为"AND"关系（同时满足）
- **实时更新**：筛选条件变更时，表格数据实时更新
- **重置功能**：支持一键重置所有筛选条件

#### 2.2.3 筛选行为差异
- **矩阵视图**：在角色级别进行筛选，如果角色包含目标模块的权限，则显示该角色的所有权限
- **列表视图**：在权限条目级别进行筛选，只显示匹配模块的具体权限条目

### 2.3 用户信息展示

#### 2.3.1 用户列显示
- **用户头像**：显示用户头像（最多显示3个）
  - 【2025年1月23日更新】显示用户名首字母的个性化头像
  - 【2025年1月23日更新】蓝色渐变背景（from-blue-400 to-blue-600）
  - 【2025年1月23日更新】白色文字，高对比度设计
- **用户计数**：显示使用该角色的用户总数
- **用户名称**：显示用户名称列表（悬停提示）
- **超量显示**：超过3个用户时显示"+N"标签，悬停显示完整用户列表
- **空状态显示**：没有用户使用该角色时显示"-"

#### 2.3.2 用户信息来源
- 基于角色分配关系，动态计算每个角色的用户列表
- 实时更新用户计数和用户信息
- 只显示当前租户下的用户信息
- 通过账号的roles字段关联角色和用户


## 3. 交互流程

### 3.1 权限查看流程

```
1. 用户进入权限查看页面
   ↓
2. 系统加载角色和账号数据
   ↓
3. 默认展示所有角色的权限
   ↓
4. 用户可以切换视图模式（按所有角色/按所有页面）
   ↓
5. 用户可以应用筛选条件（模块、角色名称）
   ↓
6. 系统根据筛选条件更新显示结果
```



### 3.2 用户信息查看流程

```
1. 用户悬停在用户头像或用户计数上
   ↓
2. 显示完整的用户名称列表
   ↓
3. 如果用户数量超过3个，显示"+N"标签
   ↓
4. 悬停在"+N"标签上显示剩余用户列表
```



### 4.3 数据处理逻辑

#### 4.3.1 矩阵视图数据处理
- **行数据**：以角色为单位组织数据
- **列数据**：以模块为单位组织权限
- **交叉数据**：计算角色在特定模块下的所有操作权限
- **用户关联**：动态计算每个角色的用户列

#### 4.3.2 筛选处理逻辑
- **角色筛选**：基于角色名称的模糊匹配
- **模块筛选**：基于权限模块的精确匹配
- **组合筛选**：多个筛选条件的AND逻辑
- **实时筛选**：输入变化时立即更新结果

## 6. 注意事项

### 6.1 业务规则

1. **数据范围**：
   - 只显示状态为ACTIVE的角色
   - 只显示有权限配置的角色（permissions.length > 0）
   - 只显示当前租户下的数据
   - 用户信息基于角色分配关系动态计算

2. **权限显示**：
   - 权限图标按照操作类型合并显示
   - 相同类型的操作只显示一个图标
   - 提供权限图例帮助用户理解
   - 空权限单元格显示"-"

3. **筛选逻辑**：
   - 矩阵视图在角色级别筛选
   - 列表视图在权限条目级别筛选
   - 筛选条件为AND关系
   - 支持实时筛选和重置功能

4. **用户信息**：
   - 基于角色分配关系动态计算
   - 实时更新用户计数
   - 保护用户隐私信息
   - 最多显示3个用户头像，超出显示"+N"

5. **视图差异**：
   - 矩阵视图适合整体权限概览
   - 列表视图适合详细权限分析
   - 两种视图数据来源一致但展示方式不同
   - 视图切换保持筛选状态
