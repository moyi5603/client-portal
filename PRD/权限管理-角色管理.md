# 权限管理 - 角色管理

## 1. 概述

### 1.1 功能定位
角色管理模块是Client Portal权限管理系统的核心功能之一，用于管理系统中的角色。主账号可以通过此模块创建、编辑、删除角色，并为角色配置细粒度的权限（模块→页面→操作），然后将角色分配给账号，实现基于角色的访问控制（RBAC）。

系统预设了超级管理员角色（Super Administrator），该角色拥有所有权限但在权限管理模块中禁用操作功能，确保系统安全。

### 1.2 业务价值
- **权限复用**：通过角色统一管理权限，避免为每个账号单独配置权限，提高管理效率
- **灵活配置**：支持细粒度的权限配置（模块→页面→操作），满足不同业务场景的权限需求
- **易于维护**：角色权限变更后，所有使用该角色的账号自动继承新权限，降低维护成本
- **权限审计**：记录角色的创建、修改、删除操作，便于权限审计和追溯
- **系统安全**：预设系统角色提供基础权限保障，同时防止权限滥用

### 1.3 目标用户
- **主账号（MAIN）**：拥有角色管理的完整权限，可以创建、编辑、删除所有角色（除系统预设角色）
- **拥有角色管理权限的账号**：通过角色分配获得角色管理权限的账号，可以创建、编辑、删除角色

## 2. 功能需求

### 2.1 系统预设角色

#### 2.1.1 超级管理员角色
- **角色ID**：ROLE-000（特殊标识）
- **角色名称**：Super Administrator
- **描述**：Super administrator with all permissions (actions disabled)
- **权限特点**：
  - 拥有所有业务模块的完整权限
  - 在权限管理模块中仅有查看权限，禁用创建、编辑、删除操作
  - 确保系统安全，防止权限滥用
- **保护机制**：
  - 标记为系统角色（isSystemRole: true）
  - 前端界面禁用编辑和删除操作
  - 后端API层面保护，拒绝修改和删除请求
- **分配对象**：系统主账号默认分配此角色

### 2.2 角色列表展示

#### 2.2.1 列表字段
- **角色名称**：角色的唯一标识（必填，唯一）
- **角色ID**：系统自动生成的角色ID（格式：ROLE-001, ROLE-002等）
- **描述**：角色的描述信息（可选）
- **模块**：显示角色配置了权限的模块列表，如果配置了所有模块则显示"全部"
- **用户数**：显示使用该角色的账号数量（可点击查看用户列表）
- **状态**：角色当前状态
  - 启用（ACTIVE）：角色可用于分配
  - 已废弃（DEPRECATED）：不允许新分配，现有用户保留访问权限
- **最后修改**：显示最后修改时间和修改者用户名
- **操作**：编辑、复制、删除按钮
- **系统角色标识**：系统预设角色显示特殊标识

#### 2.2.2 列表功能
- **筛选功能**：支持按以下条件进行筛选
  - **模块**：支持选择特定模块或"全部模块"
  - **状态**：支持选择启用、已废弃
- **搜索功能**：支持按角色名称进行搜索（只搜索角色名称）
- **组合筛选**：筛选和搜索条件可以同时使用，筛选结果为所有条件的交集
- **分页显示**：每页10条记录，支持翻页和调整每页显示数量
- **用户列表查看**：点击"用户数"可查看使用该角色的用户列表
- **系统角色保护**：系统角色的编辑和删除按钮被禁用，显示相应提示信息

### 2.2 创建角色

#### 2.2.1 必填字段
- **角色名称**：在同一租户内唯一
- **状态**：启用或已废弃，默认为"启用"

#### 2.2.2 可选字段
- **描述**：角色的描述信息
- **权限配置**：为角色配置权限（模块→页面→操作）
  - 支持按模块展开/折叠
  - 每个模块下包含多个功能页面
  - 每个功能页面支持的操作根据页面而定（如查看、创建、编辑、删除、导出等）
  - 只显示该功能页面支持的操作，不支持的操作不显示
  - 支持全选、清空操作（全局、模块级别、页面级别）

#### 2.2.3 验证规则
- 角色名称在同一租户内必须唯一
- 必填项信息完整

### 2.3 编辑角色

#### 2.3.1 可编辑字段
- **角色名称**：需验证唯一性（排除当前角色）
- **描述**：角色的描述信息
- **状态**：启用/已废弃
- **权限配置**：可以修改角色的权限配置
  - 支持按模块展开/折叠
  - 支持全选、清空操作（全局、模块级别、页面级别）
  - 权限配置变更后，使用该角色的所有账号自动继承新权限

#### 2.3.2 编辑限制
- 角色ID创建后不可修改
- 已废弃的角色不允许新分配，但现有用户保留访问权限
- 系统预设角色（如超级管理员）不允许编辑，编辑按钮被禁用并显示"系统角色只读"提示

### 2.4 删除角色

#### 2.4.1 删除确认
- 删除前显示确认对话框："确定要删除这个角色吗？"
- 确认后执行删除操作

#### 2.4.2 删除限制
- 如果角色正在被使用（用户数 > 0），删除按钮不可点击，并显示提示信息
- 系统预设角色（如超级管理员）不允许删除，删除按钮不可点击，并显示"系统角色受保护"提示
- 只能删除未被使用的非系统角色
- 删除后角色数据永久删除（需谨慎操作）

### 2.5 复制角色

#### 2.5.1 复制功能
- 点击"复制"按钮，创建新角色
- 新角色名称为原角色名称 + "（副本）"
- 新角色继承原角色的所有权限配置
- 新角色状态默认为"启用"
- 系统角色也可以被复制，但复制后的角色不再是系统角色

### 2.6 权限配置

#### 2.6.1 权限结构
- **层级结构**：模块 → 页面 → 操作
  - 模块：系统的一级功能模块
  - 页面：模块下的具体功能页面
  - 操作：页面支持的具体操作（如查看、创建、编辑、删除、导出等）
- **动态显示**：每个功能页面只显示其支持的操作，不支持的操作不显示


#### 2.6.2 权限配置操作
- **全选**：支持全局全选、模块级别全选、页面级别全选
- **清空**：支持全局清空、模块级别清空、页面级别清空
- **批量操作**：每个模块和页面都提供"全选"和"清空"快捷按钮
- **权限显示**：操作复选框在功能名称后面横向排列，间距适中

## 3. 交互流程

### 3.1 创建角色流程

```
1. 用户点击"创建角色"按钮
   ↓
2. 进入创建角色页面
   ↓
3. 用户填写角色基本信息（名称、描述、状态）
   ↓
4. 用户配置权限（选择模块→页面→操作）
   ↓
5. 用户点击"创建"按钮
   ↓
6. 验证信息（角色名称唯一性、必填项）
   ↓
7. 创建成功，返回角色列表
```

### 3.2 编辑角色流程

```
1. 用户点击角色行的"编辑"按钮
   ↓
2. 进入编辑角色页面，预填充角色信息
   ↓
3. 用户修改需要更新的字段（名称、描述、状态、权限）
   ↓
4. 用户点击"保存更改"按钮
   ↓
5. 验证信息（角色名称唯一性、必填项）
   ↓
6. 更新成功，返回角色列表
```

### 3.3 删除角色流程

```
1. 用户点击角色行的"删除"按钮
   ↓
2. 弹出确认对话框："确定要删除这个角色吗？"
   ↓
3. 用户点击"确定"
   ↓
4. 删除成功，刷新列表
```

### 3.4 复制角色流程

```
1. 用户点击角色行的"复制"按钮
   ↓
2. 系统创建新角色，名称为原角色名称 + "（副本）"
   ↓
3. 新角色继承原角色的所有权限配置
   ↓
4. 创建成功，刷新列表
```

### 3.5 查看用户列表流程

```
1. 用户点击角色行的"用户数"（数字）
   ↓
2. 弹出用户列表Modal
   ↓
3. 显示使用该角色的所有用户信息（用户名、邮箱、手机号、账号类型、状态）
   ↓
4. 用户点击"关闭"按钮关闭Modal
```

## 5. 注意事项

### 5.1 业务规则

1. **角色ID自动生成**：角色ID由系统自动生成，格式为ROLE-001, ROLE-002等，创建后不可修改
2. **角色状态**：
   - 启用（ACTIVE）：角色可用于分配
   - 已废弃（DEPRECATED）：不允许新分配，现有用户保留访问权限
3. **删除限制**：
   - 如果角色正在被使用（用户数 > 0），无法删除
   - 删除前需要先移除所有使用该角色的账号
4. **权限配置**：
   - 权限配置采用模块→页面→操作的层级结构
   - 每个功能页面只显示其支持的操作
   - 权限配置变更后，使用该角色的所有账号自动继承新权限
5. **唯一性验证**：角色名称在同一租户内必须唯一
6. **系统角色保护**：
   - 系统预设角色（如超级管理员）标记为isSystemRole: true
   - 系统角色不允许编辑或删除，前后端都有保护机制
   - 系统角色可以被复制，但复制后的角色不再是系统角色
7. **权限分离原则**：
   - 超级管理员角色拥有所有业务权限
   - 在权限管理模块中仅有查看权限，防止权限滥用
