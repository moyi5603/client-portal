# 权限管理 - 操作记录

## 1. 概述

### 1.1 功能定位
操作记录模块是Client Portal权限管理系统的审计功能，用于记录和查看系统中的所有关键操作。主账号可以通过此模块查看账号管理和角色管理相关的操作记录，包括创建、编辑、删除、复制等操作的详细信息，为系统安全审计和问题追溯提供数据支持。

### 1.2 业务价值
- **操作审计**：完整记录系统中的关键操作，满足安全审计要求
- **问题追溯**：当出现问题时，可以快速定位相关操作和责任人
- **合规要求**：满足企业内控和合规审计的要求
- **安全监控**：通过操作记录监控异常操作，提高系统安全性
- **操作透明**：让管理员了解系统中发生的所有变更操作

### 1.3 目标用户
- **主账号（MAIN）**：拥有操作记录查看的完整权限，可以查看所有操作记录
- **拥有操作记录查看权限的账号**：通过角色分配获得操作记录查看权限的账号，可以查看操作记录

## 2. 功能需求

### 2.1 操作记录列表展示

#### 2.1.1 列表字段
- **操作时间**：操作发生的时间（格式：YYYY-MM-DD HH:mm:ss）
- **操作者**：执行操作的用户名
- **操作类型**：操作的类型标签
  - 创建账号
  - 编辑账号
  - 删除账号
  - 创建角色
  - 编辑角色
  - 复制角色
  - 删除角色
- **操作对象**：被操作的对象信息
  - 主标题：对象名称（用户名或角色名）
  - 副标题：对象类型标签 + 对象ID
    - 账号类操作：显示"账号"标签 + 账号ID（如ACC-001）
    - 角色类操作：显示"角色"标签 + 角色ID（如ROLE-001）
- **描述**：操作的详细描述，支持换行显示
  - 创建操作：显示完整的创建信息
  - 编辑操作：显示具体的修改内容（修改前后对比）
  - 删除操作：显示被删除对象的基本信息
  - 复制操作：显示源对象和新对象的信息

#### 2.1.2 列表功能
- **筛选功能**：支持按以下条件进行筛选
  - **操作日期**：支持选择日期范围
  - **操作类型**：支持选择特定操作类型或"全部"
  - **对象类型**：支持选择账号、角色或"全部"
  - **操作者**：支持按操作者用户名进行模糊搜索
  - **操作对象**：支持按操作对象名称进行模糊搜索
- **组合筛选**：多个筛选条件可以同时使用，筛选结果为所有条件的交集
- **重置功能**：支持一键重置所有筛选条件
- **分页显示**：每页20条记录，支持翻页和调整每页显示数量
- **时间排序**：记录按操作时间倒序排列（最新的在前）

### 2.2 操作描述规则

#### 2.2.1 创建账号描述
**格式**：
```
创建账号：<用户名>
账号类型：<账号类型>
邮箱：<邮箱>
手机号：<手机号>
状态：<状态>
可访问customer：<customer1、customer2、...>
角色：<角色1、角色2、...>
```

**示例**：
```
创建账号：zhangsan
账号类型：客户子账号
邮箱：123@qq.com
手机号：12345678901
状态：启用
可访问customer：customer1、customer2、customer3
角色：角色1、角色2、角色3
```

#### 2.2.2 编辑账号描述
**格式**：
```
编辑账号：
修改<字段名>由<修改前内容>改为<修改后内容>
修改<字段名>由<修改前内容>改为<修改后内容>
...
```

**示例**：
```
编辑账号：
修改<邮箱>由<123@qq.com>改为<345@qq.com>
修改<状态>由<启用>改为<禁用>
```

#### 2.2.3 删除账号描述
**格式**：
```
删除账号：<用户名>
```

**示例**：
```
删除账号：zhangsan
```

#### 2.2.4 创建角色描述
**格式**：
```
创建角色：<角色名称>（<角色ID>）
描述：<描述>
状态：<状态>
权限配置：
<一级菜单-二级菜单（操作1、操作2、...）>
<一级菜单-二级菜单（操作1、操作2、...）>
...
```

**示例**：
```
创建角色：操作员A（ROLE-099）
描述：这是一段描述
状态：启用
权限配置：
Dashboards-KPI（查看）
Inbound-Inquiry（查看、编辑、导出）
Inbound-Appointment List（查看、创建、Cancel）
```

#### 2.2.5 编辑角色描述
**格式**：
```
编辑角色：
修改<字段名>由<修改前内容>改为<修改后内容>
修改<权限配置><一级菜单-二级菜单>由<操作1、操作2>改为<操作1、操作3>
...
```

**示例**：
```
编辑角色：
修改<角色名称>由<操作员A>改为<操作员B>
修改<权限配置>Inbound-Inquiry由<查看、编辑>改为<查看、导出>
修改<权限配置>Inbound-Schedule Summary由<查看>改为<无>
```

#### 2.2.6 复制角色描述
**格式**：
```
复制角色：<角色名称>（<角色ID>），创建角色：<角色名称>（<角色ID>）
```

**示例**：
```
复制角色：操作员A（ROLE-099），创建角色：操作员A（副本）（ROLE-105）
```

#### 2.2.7 删除角色描述
**格式**：
```
删除角色：<角色名称>（<角色ID>）
```

**示例**：
```
删除角色：操作员A（ROLE-099）
```

### 2.3 筛选和搜索功能

#### 2.3.1 筛选条件
- **操作日期筛选**：支持选择日期范围
  - 使用日期选择器，不包含具体时间
  - 支持选择开始日期和结束日期
  - 筛选逻辑：包含选择日期范围内的所有操作记录
- **操作类型筛选**：支持选择特定操作类型
  - 下拉选择框，包含所有操作类型
  - 默认选择"全部"
  - 筛选后只显示选中类型的操作记录
- **对象类型筛选**：支持选择账号、角色
  - 下拉选择框，包含"账号"、"角色"、"全部"
  - 默认选择"全部"
  - 筛选后只显示选中对象类型的操作记录
- **操作者搜索**：支持按操作者用户名搜索
  - 文本输入框，支持模糊搜索
  - 不区分大小写的模糊匹配
  - 支持清空功能
- **操作对象搜索**：支持按操作对象名称搜索
  - 文本输入框，支持模糊搜索
  - 不区分大小写的模糊匹配
  - 支持清空功能

#### 2.3.2 筛选逻辑
- **组合筛选**：所有筛选条件可以同时使用
- **筛选关系**：多个筛选条件为"AND"关系（同时满足）
- **实时筛选**：筛选条件变更时，点击"搜索"按钮应用筛选
- **重置功能**：支持一键重置所有筛选条件

### 2.4 数据展示和分页

#### 2.4.1 表格功能
- **列宽优化**：合理分配各列宽度，避免右侧空白过多
  - 操作时间：160px
  - 操作者：120px
  - 操作类型：100px
  - 操作对象：180px
  - 描述：自适应剩余空间
- **换行处理**：描述列支持自动换行显示
  - 使用`whiteSpace: 'pre-wrap'`支持换行符
  - 设置合适的行高提高可读性
  - 长文本自动换行，避免横向滚动
- **水平滚动**：支持水平滚动查看完整内容
- **加载状态**：数据加载时显示加载动画
- **空状态处理**：没有数据时显示友好的空状态提示

#### 2.4.2 分页设置
- **每页记录数**：默认每页20条记录
- **分页控制**：支持调整每页显示数量（10/20/50/100）
- **分页信息**：显示当前页范围和总记录数
- **页面跳转**：支持页面跳转功能

### 2.5 系统字段过滤

#### 2.5.1 过滤规则
系统自动过滤以下内部字段的变更，不在操作描述中显示：
- **updatedAt**：更新时间戳
- **createdAt**：创建时间戳
- **modifiedBy**：修改者ID
- **createdBy**：创建者ID
- **usageCount**：使用计数
- **id**：记录ID
- **tenantId**：租户ID

#### 2.5.2 过滤目的
- **用户友好**：只显示用户关心的业务字段变更
- **简洁明了**：避免技术性字段干扰用户理解
- **关注业务**：突出业务相关的重要变更信息

## 3. 交互流程

### 3.1 查看操作记录流程

```
1. 用户进入操作记录页面
   ↓
2. 系统加载操作记录数据
   ↓
3. 默认显示所有操作记录，按时间倒序排列
   ↓
4. 用户可以应用筛选条件（日期、类型、对象类型、操作者、操作对象）
   ↓
5. 系统根据筛选条件更新显示结果
```

### 3.2 筛选操作流程

```
1. 用户设置筛选条件
   ├─ 选择操作日期范围
   ├─ 选择操作类型
   ├─ 选择对象类型
   ├─ 输入操作者搜索
   └─ 输入操作对象搜索
   ↓
2. 用户点击"搜索"按钮
   ↓
3. 系统应用筛选条件，更新显示结果
   ↓
4. 用户可以点击"重置"按钮清空所有筛选条件
```

### 3.3 分页操作流程

```
1. 用户查看当前页的操作记录
   ↓
2. 用户可以：
   ├─ 点击页码跳转到指定页面
   ├─ 使用上一页/下一页按钮翻页
   └─ 调整每页显示数量
   ↓
3. 系统更新显示对应页面的数据
```

## 5. 注意事项

### 5.1 业务规则

1. **数据范围**：
   - 只显示当前租户下的操作记录
   - 记录按操作时间倒序排列（最新的在前）
   - 支持大数据量的分页显示

2. **操作记录类型**：
   - 账号相关：创建账号、编辑账号、删除账号
   - 角色相关：创建角色、编辑角色、复制角色、删除角色

5. **系统字段过滤**：
