# 权限管理 - 菜单管理

## 1. 概述

### 1.1 功能定位
菜单管理模块是Portal Admin权限管理系统的核心功能之一，用于管理系统中的菜单结构。Portal Admin可以通过此模块创建、编辑、删除菜单项，配置菜单的层级关系、显示顺序和状态，为系统构建灵活的导航结构。

菜单支持三种类型：目录（Directory）、菜单（Menu）、按钮（Button），形成三级层级结构，满足不同的导航和权限控制需求。

### 1.2 业务价值
- **灵活导航**：支持多层级菜单结构，满足复杂的业务导航需求
- **权限控制**：通过菜单配置实现细粒度的功能访问控制
- **易于维护**：集中管理菜单结构，便于系统功能的组织和调整
- **用户体验**：通过合理的菜单组织提升用户操作效率
- **国际化支持**：菜单名称支持中英文显示，提升国际化体验

### 1.3 目标用户
- **Portal Admin**：拥有菜单管理的完整权限，可以创建、编辑、删除所有菜单项

## 2. 功能需求

### 2.1 菜单列表展示

#### 2.1.1 列表字段
- **名称（Name）**：菜单的显示名称
  - 支持中英文翻译显示
  - 树形结构展示，通过缩进显示层级关系
  - 支持展开/收起子菜单
- **代码（Code）**：菜单的唯一标识代码
  - 目录类型不显示代码
  - 菜单和按钮类型显示代码（如：menu-1, button-1）
- **类型（Type）**：菜单类型标识
  - 目录（DIRECTORY）：蓝色标签
  - 菜单（MENU）：绿色标签
  - 按钮（BUTTON）：黄色标签
- **显示顺序（Order）**：菜单的排序序号
- **状态（Status）**：菜单当前状态
  - 启用（Enabled）：绿色标签，对应后端NORMAL状态
  - 禁用（Disabled）：灰色标签，对应后端DISABLED状态
- **操作（Actions）**：操作按钮（...菜单）
  - 添加（Add）：添加子菜单（按钮类型不显示）
  - 编辑（Edit）：编辑当前菜单
  - 删除（Delete）：删除当前菜单

#### 2.1.2 列表功能
- **树形结构**：以树形结构展示菜单层级关系
  - 支持展开/收起操作
  - 默认状态：所有菜单收起
  - 通过缩进显示层级关系（每级缩进20px）
  - 展开/收起图标：ChevronDown（展开）/ ChevronRight（收起）
- **中英文翻译**：菜单名称自动翻译为英文显示
  - 工作台 → Dashboard
  - 采购管理 → Purchase Management
  - 订单管理 → Order Management
  - 运输管理 → Transportation Management
  - 等等（详见翻译映射表）
- **操作菜单**：每行提供...按钮，点击显示操作菜单
  - 添加：仅对目录和菜单类型显示
  - 编辑：所有类型都显示
  - 删除：所有类型都显示

### 2.2 创建菜单

#### 2.2.1 必填字段
- **类型（Type）**：目录、菜单、按钮（单选，横向排列）
- **菜单名称（Menu Name）**：菜单的显示名称
- **菜单代码（Menu Code）**：菜单的唯一标识（目录类型不需要）
- **状态（Status）**：启用或禁用（单选，横向排列）

#### 2.2.2 可选字段
- **父菜单（Parent Menu）**：选择父级菜单
  - 下拉选择框，支持树形结构展开/收起
  - 默认选项：None (Root Level)
  - 只显示目录和菜单类型（不显示按钮类型）
  - 支持中英文翻译显示
- **显示顺序（Display Order）**：数字输入框，默认为0

#### 2.2.3 验证规则
- 菜单名称必填
- 菜单代码在非目录类型时必填
- 菜单代码格式验证（如：menu-1, button-1）

### 2.3 编辑菜单

#### 2.3.1 可编辑字段
- **类型（Type）**：可修改（单选，横向排列）
- **父菜单（Parent Menu）**：可修改
- **菜单名称（Menu Name）**：可修改
- **菜单代码（Menu Code）**：可修改（目录类型不显示）
- **显示顺序（Display Order）**：可修改
- **状态（Status）**：可修改（单选，横向排列）

#### 2.3.2 编辑行为
- 编辑时弹出对话框，预填充当前菜单信息
- 状态值自动转换：
  - 后端NORMAL → 前端Enabled
  - 后端DISABLED → 前端Disabled
  - 前端Enabled → 后端NORMAL
  - 前端Disabled → 后端DISABLED
- 父菜单选择器默认展开所有可选项

### 2.4 删除菜单

#### 2.4.1 删除确认
- 删除前显示确认对话框
- 显示要删除的菜单名称（翻译后的英文名称）
- 确认后执行删除操作

#### 2.4.2 删除限制
- 删除后菜单数据永久删除（需谨慎操作）
- 如果菜单有子菜单，需要先删除子菜单

### 2.5 菜单类型说明

#### 2.5.1 目录（DIRECTORY）
- **用途**：用于组织菜单结构，不对应具体功能页面
- **特点**：
  - 不需要菜单代码
  - 可以包含子菜单（目录、菜单、按钮）
  - 支持添加子菜单操作
- **示例**：采购管理、订单管理、运输管理等

#### 2.5.2 菜单（MENU）
- **用途**：对应具体的功能页面
- **特点**：
  - 需要菜单代码（如：menu-1）
  - 可以包含子菜单（按钮）
  - 支持添加子菜单操作
- **示例**：账号管理、角色管理、权限查看等

#### 2.5.3 按钮（BUTTON）
- **用途**：对应页面内的具体操作按钮
- **特点**：
  - 需要菜单代码（如：button-1）
  - 不能包含子菜单
  - 不显示添加子菜单操作
- **示例**：创建、编辑、删除、导出等操作按钮

## 3. 交互流程

### 3.1 创建菜单流程

```
1. 用户点击"Add Menu"按钮
   ↓
2. 弹出创建菜单对话框
   ↓
3. 用户选择菜单类型（目录/菜单/按钮）
   ↓
4. 用户选择父菜单（可选）
   ↓
5. 用户填写菜单名称
   ↓
6. 用户填写菜单代码（非目录类型）
   ↓
7. 用户设置显示顺序
   ↓
8. 用户选择状态（启用/禁用）
   ↓
9. 用户点击"Create"按钮
   ↓
10. 验证信息（必填项、格式）
   ↓
11. 创建成功，关闭对话框，刷新列表
```

### 3.2 编辑菜单流程

```
1. 用户点击菜单行的"..."按钮
   ↓
2. 点击"Edit"选项
   ↓
3. 弹出编辑菜单对话框，预填充菜单信息
   ↓
4. 用户修改需要更新的字段
   ↓
5. 用户点击"Update"按钮
   ↓
6. 验证信息（必填项、格式）
   ↓
7. 更新成功，关闭对话框，刷新列表
```

### 3.3 删除菜单流程

```
1. 用户点击菜单行的"..."按钮
   ↓
2. 点击"Delete"选项
   ↓
3. 弹出确认对话框，显示要删除的菜单名称
   ↓
4. 用户点击"Delete"确认
   ↓
5. 删除成功，刷新列表
```

### 3.4 添加子菜单流程

```
1. 用户点击菜单行的"..."按钮
   ↓
2. 点击"Add"选项（仅目录和菜单类型显示）
   ↓
3. 弹出创建菜单对话框
   ↓
4. 父菜单自动设置为当前菜单
   ↓
5. 用户填写其他信息
   ↓
6. 创建成功，刷新列表
```

### 3.5 展开/收起菜单流程

```
1. 用户点击菜单行的展开/收起图标
   ↓
2. 如果当前是收起状态，展开显示子菜单
   ↓
3. 如果当前是展开状态，收起隐藏子菜单
   ↓
4. 更新展开状态，重新渲染列表
```

## 4. 数据模型

### 4.1 菜单数据结构

```typescript
interface Menu {
  id: string;                    // 菜单ID（UUID）
  name: string;                  // 菜单名称
  code?: string;                 // 菜单代码（目录类型可选）
  path?: string;                 // 路由路径（可选）
  parentId?: string;             // 父菜单ID（可选）
  icon?: string;                 // 图标（可选）
  order: number;                 // 显示顺序
  type: 'DIRECTORY' | 'MENU' | 'BUTTON';  // 菜单类型
  isExternal?: boolean;          // 是否外链（可选）
  visible?: boolean;             // 显示状态（可选）
  status: 'NORMAL' | 'DISABLED'; // 菜单状态
  componentPath?: string;        // 组件路径（可选）
  routeParams?: string;          // 路由参数（可选）
  children?: Menu[];             // 子菜单列表
}
```

### 4.2 菜单类型枚举

```typescript
enum MenuType {
  DIRECTORY = 'DIRECTORY',  // 目录
  MENU = 'MENU',           // 菜单
  BUTTON = 'BUTTON'        // 按钮
}
```

### 4.3 菜单状态枚举

```typescript
enum MenuStatus {
  NORMAL = 'NORMAL',       // 启用（后端）
  DISABLED = 'DISABLED'    // 禁用（后端）
}
```

## 5. API接口

### 5.1 获取菜单列表

**接口**：`GET /api/menus`

**请求参数**：无

**响应数据**：
```json
{
  "success": true,
  "data": [
    {
      "id": "uuid-1",
      "name": "工作台",
      "code": null,
      "path": "/dashboard",
      "parentId": null,
      "order": 1,
      "type": "DIRECTORY",
      "status": "NORMAL",
      "children": [...]
    }
  ]
}
```

### 5.2 创建菜单

**接口**：`POST /api/menus`

**请求参数**：
```json
{
  "name": "菜单名称",
  "code": "menu-1",
  "path": "/path",
  "type": "MENU",
  "parentId": "parent-uuid",
  "order": 1,
  "status": "NORMAL"
}
```

**响应数据**：
```json
{
  "success": true,
  "data": {
    "id": "new-uuid",
    "name": "菜单名称",
    ...
  }
}
```

### 5.3 更新菜单

**接口**：`PUT /api/menus/:id`

**请求参数**：
```json
{
  "name": "更新后的名称",
  "code": "menu-1",
  "path": "/path",
  "type": "MENU",
  "parentId": "parent-uuid",
  "order": 2,
  "status": "DISABLED"
}
```

**响应数据**：
```json
{
  "success": true,
  "data": {
    "id": "uuid",
    "name": "更新后的名称",
    ...
  }
}
```

### 5.4 删除菜单

**接口**：`DELETE /api/menus/:id`

**请求参数**：无

**响应数据**：
```json
{
  "success": true,
  "message": "Menu deleted successfully"
}
```

## 7. 注意事项

### 7.1 业务规则

1. **菜单层级**：
   - 支持多层级菜单结构
   - 目录可以包含目录、菜单、按钮
   - 菜单可以包含按钮
   - 按钮不能包含子菜单

2. **菜单代码**：
   - 目录类型不需要菜单代码
   - 菜单和按钮类型必须有菜单代码
   - 菜单代码用于权限控制和功能映射

3. **状态管理**：
   - 后端使用NORMAL/DISABLED
   - 前端显示Enabled/Disabled
   - 需要在前后端之间进行状态转换

4. **默认状态**：
   - 页面加载时所有菜单默认收起
   - 创建菜单时状态默认为启用
   - 显示顺序默认为0

5. **中英文翻译**：
   - 菜单名称在前端自动翻译为英文
   - 数据库存储中文名称
   - 翻译映射在前端维护

6. **删除限制**：
   - 删除菜单前需要先删除子菜单
   - 删除操作不可恢复，需谨慎操作
